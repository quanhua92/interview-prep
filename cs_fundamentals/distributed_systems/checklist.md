# Distributed Systems — Self-Assessment Checklist

Use this checklist to gauge your readiness for distributed systems interview questions. Check off items as you can confidently explain them without notes.

## Concept Self-Assessment

- [ ] Can you explain the CAP theorem and how it applies to real-world system design trade-offs?
- [ ] Can you describe consistency models, including strong, eventual, causal, and linearizable consistency?
- [ ] Can you explain consensus algorithms such as Paxos and Raft at a high level?
- [ ] Can you describe common fault tolerance techniques like retries, circuit breakers, and bulkheads?
- [ ] Can you explain how consistent hashing works and why it minimizes data movement when nodes change?
- [ ] Can you describe the gossip protocol and how it is used for membership and failure detection?
- [ ] Can you explain partitioning (sharding) strategies and their trade-offs?
- [ ] Can you describe the difference between idempotent and non-idempotent operations and why it matters?

## Explain-Out-Loud Prompts

- [ ] Explain how you would design a distributed cache that stays consistent across multiple nodes.
- [ ] Walk through what happens when a network partition splits your database cluster in two.
- [ ] Describe how a message queue helps decouple services and handle failure scenarios.
- [ ] Explain how gossip protocol propagates information in a decentralized system and why it is eventually consistent.

## Common Questions Practice

- [ ] What is the difference between vertical and horizontal scaling, and when would you choose each?
- [ ] Explain the leader-follower pattern and how leader election works.
- [ ] How does replication work, and what are eventual consistency vs. strong consistency trade-offs?
- [ ] What is a vector clock and how does it help detect causality in distributed systems?
- [ ] How would you handle distributed transactions across multiple services?
- [ ] Explain the Byzantine Generals Problem and why it matters for system design.
