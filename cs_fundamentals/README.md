# CS Fundamentals

Concept-focused preparation for computer science fundamentals interviews. Each topic has:

- **discussion.md** — Key concepts, definitions, common questions with answer outlines, quick reference tables
- **checklist.md** — Self-assessment prompts, explain-out-loud exercises, and practice log

## Topics

| Topic | Focus |
|-------|-------|
| [Operating Systems](operating_systems/) | Processes, threads, memory, file systems |
| [Computer Networking](computer_networking/) | TCP/IP, HTTP, DNS, TLS, network models |
| [Databases](databases/) | SQL, indexing, transactions, ACID, normalization |
| [Concurrency](concurrency/) | Threads, locks, race conditions, deadlocks |
| [Distributed Systems](distributed_systems/) | CAP, consensus, replication, fault tolerance |
| [Data Structures & Algos](data_structures_algos/) | Arrays, trees, graphs, hash tables, complexity |
| [System Security](system_security/) | OWASP, auth, encryption, secure design |
| [API Security](api_security/) | API auth (OAuth2, JWT, mTLS), authorization (RBAC, ABAC, BOLA), LLM API key management, rate limiting, OWASP API Top 10, CORS, CSRF, security headers, secrets management |
| [Data Pipelines](data_pipelines/) | ETL/ELT, streaming, orchestration, feature stores, lakehouse formats, AI/ML pipelines |

## How to Practice

1. Read the **discussion.md** once to refresh your understanding
2. Open the **checklist.md** and work through the self-assessment prompts
3. Practice explaining each concept out loud in 60-90 seconds
4. For each "Can you explain X?" prompt, give a clear, concise answer without notes
5. Use the practice log to track which concepts need more review
