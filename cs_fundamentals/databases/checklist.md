# Databases — Self-Assessment Checklist

Use this checklist to gauge your readiness for database-related interview questions. Check off items as you can confidently explain them without notes.

## Concept Self-Assessment

- [ ] Can you explain ACID properties and why each one matters for transaction correctness?
- [ ] Can you compare SQL and NoSQL databases, including when you would choose each?
- [ ] Can you describe how database indexing works, including B-trees, hash indexes, and trade-offs?
- [ ] Can you explain the different isolation levels and which anomalies each prevents?
- [ ] Can you describe normalization forms (1NF through 3NF) and when denormalization is appropriate?
- [ ] Can you explain how query optimization works, including the role of the query planner and execution plan?

## Explain-Out-Loud Prompts

- [ ] Explain how you would design a database schema for a social media application like Twitter.
- [ ] Walk through how a SELECT query is executed internally, from parsing to returning results.
- [ ] Describe how you would diagnose and resolve slow query performance in a production database.
- [ ] Explain how a database manages connections, and why connection pooling improves application performance.

## Common Questions Practice

- [ ] What is the difference between a clustered and a non-clustered index?
- [ ] Explain the CAP theorem and give an example of a system that prioritizes each pair.
- [ ] Explain the difference between optimistic and pessimistic locking.
- [ ] What are the different types of SQL joins (INNER, LEFT, RIGHT, FULL, CROSS) and when would you use each?
