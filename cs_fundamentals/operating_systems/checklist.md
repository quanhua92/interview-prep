# Operating Systems — Self-Assessment Checklist

Use this checklist to gauge your readiness for OS-related interview questions. Check off items as you can confidently explain them without notes.

## Concept Self-Assessment

- [ ] Can you explain the difference between processes and threads, and when you would use one over the other?
- [ ] Can you describe how virtual memory works, including page tables, TLBs, and page replacement algorithms?
- [ ] Can you explain what a context switch is and why it is expensive?
- [ ] Can you compare deadlock prevention, avoidance, detection, and recovery strategies?
- [ ] Can you explain how the operating system handles interrupts and system calls?
- [ ] Can you describe the difference between user space and kernel space and why the distinction matters?
- [ ] Can you explain demand paging, page replacement algorithms (LRU, FIFO, Clock), and Belady's anomaly?
- [ ] Can you describe the fork/exec/wait lifecycle and how processes are created and reaped?

## Explain-Out-Loud Prompts

- [ ] Explain to a junior developer what happens when you type a URL into the browser from the OS perspective.
- [ ] Walk through how a process is created, scheduled, and terminated in a modern OS.
- [ ] Describe how memory fragmentation occurs and how paging and segmentation address it.
- [ ] Compare LRU and FIFO page replacement algorithms, and explain why FIFO can exhibit Belady's anomaly.

## Common Questions Practice

- [ ] Explain the producer-consumer problem and how semaphores solve it.
- [ ] What is the difference between starvation and deadlock? Give real-world analogies for each.
- [ ] How does the OS decide which process to run next? Explain at least two scheduling algorithms.
- [ ] What is a zombie process and how can it be prevented?
- [ ] Explain the difference between monolithic kernels and microkernels, and trade-offs of each.
- [ ] Explain the fork/exec/wait lifecycle and why each step is necessary.
