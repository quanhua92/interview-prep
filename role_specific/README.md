# Role-Specific Preparation

Focused preparation for common engineering roles, covering core competencies and interview topics unique to each position. Each topic has:

- **discussion.md** — Core competencies table, common interview topics, key terminology, cross-references
- **checklist.md** — Topic prep checklist, practice questions, system design bridge items, and practice log

## Topics

| Topic | Focus |
|-------|-------|
| [Backend Engineer](backend_engineer/) | APIs, databases, services, scalability |
| [Frontend Engineer](frontend_engineer/) | DOM, frameworks, performance, accessibility |
| [Data Engineer](data_engineer/) | ETL, data pipelines, warehousing, streaming |
| [ML/AI Engineer](ml_ai_engineer/) | ML fundamentals, deployment, evaluation |
| [DevOps/SRE](devops_sre/) | CI/CD, containers, monitoring, reliability |
| [Full-Stack Engineer](full_stack_engineer/) | Frontend + backend breadth, system integration |
| [Senior AI Engineer](senior_ai_engineer/) | Production GenAI systems, RAG, agents, evaluation, deployment |
| [Screening & Hiring AI Engineers](screening_hiring_ai_engineers/) | Interview process design, take-home assignments, scoring rubrics, red/green flags, builders vs. talkers |

## How to Practice

1. Read the **discussion.md** for your target role(s)
2. Open the **checklist.md** and assess your readiness per competency
3. Work through the practice questions — answer out loud
4. Bridge system design topics to your role's specific concerns
5. Cross-reference with system_design and cs_fundamentals sections as needed
