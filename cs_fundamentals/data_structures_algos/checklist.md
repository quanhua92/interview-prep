# Data Structures & Algorithms — Self-Assessment Checklist

Use this checklist to gauge your readiness for DSA-related interview questions. Check off items as you can confidently explain them without notes.

## Concept Self-Assessment

- [ ] Can you explain the time and space complexity trade-offs of arrays, linked lists, hash maps, and trees?
- [ ] Can you describe when to use BFS vs. DFS and walk through each on a sample graph?
- [ ] Can you explain dynamic programming and identify overlapping subproblems in a given problem?
- [ ] Can you describe how a hash table handles collisions, including chaining and open addressing?
- [ ] Can you explain the properties of a binary search tree, AVL tree, and red-black tree, and their use cases?
- [ ] Can you describe the difference between stable and unstable sorting algorithms and name examples of each?
- [ ] Can you explain what a Trie (prefix tree) is, its use cases (autocomplete, spell-check, IP routing), and its time/space complexity?

## Explain-Out-Loud Prompts

- [ ] Explain how you would find the shortest path between two nodes in a weighted graph.
- [ ] Walk through your thought process for solving "two sum" using different approaches with trade-offs.
- [ ] Describe how an LRU cache works and implement one using the optimal data structure combination.

## Common Questions Practice

- [ ] What is the time complexity of inserting into and searching a binary search tree in the worst case?
- [ ] Explain quicksort and mergesort, including their best, average, and worst-case complexities.
- [ ] How would you detect a cycle in a linked list? Describe multiple approaches.
- [ ] What is a heap and how is it used in a priority queue?
- [ ] Explain topological sort and give a real-world application for it.
