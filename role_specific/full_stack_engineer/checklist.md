# Full-Stack Engineer -- Prep Checklist

Use this checklist to track your preparation progress across core full-stack engineering competencies. Aim to complete every item before your interview window opens.

## Topic Prep Checklist per Competency

- [ ] **Frontend Fundamentals**: Review HTML semantics, CSS layout (flexbox, Grid), JavaScript event loop, closures, and DOM manipulation.
- [ ] **Frontend Framework**: Be prepared to explain component lifecycle, state management, hooks, and performance optimization in React, Vue, or Angular.
- [ ] **Backend API Development**: Practice building RESTful APIs, handling authentication, input validation, error handling, and middleware patterns.
- [ ] **Database Design**: Review relational database design (normalization, indexing), basic SQL queries, and ORM usage; understand when to use NoSQL.
- [ ] **Authentication and Security**: Understand OAuth 2.0, JWT, session management, CSRF protection, XSS prevention, and CORS configuration.
- [ ] **DevOps Basics**: Be familiar with Docker for containerization, basic CI/CD pipelines, and deploying applications to cloud platforms.
- [ ] **System Integration**: Practice connecting frontend to backend, handling API versioning, file uploads, real-time communication (WebSockets), and third-party integrations.
- [ ] **Testing Across the Stack**: Write unit tests for both frontend and backend, integration tests for APIs, and end-to-end tests (Cypress, Playwright); understand testing strategy across the full stack.
- [ ] **Trade-off Awareness**: Be prepared to discuss depth vs breadth, when to specialize vs generalize, rapid prototyping vs production readiness, and pragmatic technology choices.
- [ ] **Architecture Decisions**: Understand monolith vs microservices trade-offs, repo structure (monorepo vs separate repos), and when to reach for API gateways and BFF patterns.
- [ ] **Key Full-Stack Terminology**: Be familiar with BFF (Backend for Frontend), feature flags, optimistic updates, API gateways, and environment parity concepts.

## Practice Questions

1. How would you design a full-stack application architecture for a task management tool like Trello?
2. Explain how you would implement authentication from the frontend login form through to the backend API.
3. How do you handle state management in a frontend application that needs to share data across many components?
4. Describe how you would optimize a full-stack application that is slow to load on mobile devices.
5. How would you design a real-time chat feature using WebSockets?
6. Explain the difference between server-side rendering and client-side rendering. When would you choose each?
7. How do you structure a REST API to support filtering, sorting, and pagination?
8. What is your approach to error handling across the full stack (frontend, API, database)?
9. How would you implement file upload and storage for user profile images?
10. Describe how you would set up a CI/CD pipeline for a full-stack application.
11. How do you secure a web application against common vulnerabilities like XSS, CSRF, and SQL injection?
12. How do you approach testing strategy across the full stack (unit, integration, E2E)?
13. When would you choose a monolith vs microservices for a new project?
14. How do you manage database migrations in a project with multiple developers?
15. How do you structure your repository -- monorepo vs separate repos, and what are the trade-offs?
