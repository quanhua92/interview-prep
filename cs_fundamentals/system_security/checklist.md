# System Security — Self-Assessment Checklist

Use this checklist to gauge your readiness for security-related interview questions. Check off items as you can confidently explain them without notes.

## Concept Self-Assessment

- [ ] Can you explain the OWASP Top 10 and describe how to mitigate at least five of the listed vulnerabilities?
- [ ] Can you describe how authentication differs from authorization, and common approaches for each?
- [ ] Can you explain the difference between symmetric and asymmetric encryption and when each is used?
- [ ] Can you describe how HTTPS works, including the role of TLS handshakes and certificates?
- [ ] Can you explain common attack vectors such as SQL injection, XSS, CSRF, and how to prevent them?
- [ ] Can you describe the principle of least privilege and how it applies to system design?

## Explain-Out-Loud Prompts

- [ ] Explain how you would secure a REST API, including authentication, input validation, and rate limiting.
- [ ] Walk through how a man-in-the-middle attack works and how TLS prevents it.
- [ ] Describe how you would store user passwords securely in a database and handle password reset flows.

## Common Questions Practice

- [ ] What is the difference between HMAC and digital signatures?
- [ ] Explain how OAuth 2.0 works, the difference between access tokens and refresh tokens, and the role of JWT in token-based authentication.
- [ ] What is DDoS (Distributed Denial of Service) and what mitigation strategies exist (rate limiting, CDN, WAF, anycast)?
- [ ] What is clickjacking and how do X-Frame-Options and CSP frame-ancestors prevent it?
- [ ] What are CORS and how do same-origin policies protect users?
- [ ] How does a session-based authentication system differ from a token-based one?
- [ ] What is a salt in password hashing and why is it important?
